# CodeAtlas 2–3 Minute Demo Script

Target duration: **2 minutes 35 seconds**. Record at 1080p, browser zoom 100%, with backend, frontend, Ollama, and one project pre-indexed before recording.

## Before recording

- Start Ollama and confirm `phi3:latest` with `ollama list`.
- Start the backend and frontend.
- Import one visually interesting public repository.
- Ask one AI question before recording so the embedding index and model are warm.
- Return to the landing page.
- Close notifications and unrelated applications.

## Narration and clicks

### 0:00–0:20 — Problem

**Show:** Hero and privacy claims.

**Say:**

> Joining an unfamiliar codebase means hunting through folders, manifests, and entry points before useful work can begin. Cloud assistants can also expose private source code. CodeAtlas is an on-device AI software architect that turns a repository into an explorable local map.

### 0:20–0:42 — Import and processing

**Show:** Click **Upload Repository**, show GitHub URL import and the Local folder alternative. Use a prepared public URL. Briefly show scan progress; cut waiting time if necessary.

**Say:**

> I can import a public GitHub repository or choose a local folder. CodeAtlas copies it into a localhost workspace, ignores generated and sensitive paths, scans real files, extracts supported symbols, and detects architecture using scored evidence.

### 0:42–1:05 — Architecture evidence

**Show:** Open dashboard → Architecture. Hover or point at detected cards and confidence/evidence.

**Say:**

> These results are not an LLM guess. The detector combines manifests, imports, filenames, folders, and known framework signals, then shows its confidence and matched evidence.

### 1:05–1:30 — 3D Galaxy

**Show:** Galaxy. Rotate, select a planet, focus it, and show real file/language/size data.

**Say:**

> The scan becomes an interactive 3D galaxy. Every planet is generated from an actual top-level project area, with real file counts, language, and size. Current links represent containment; dependency extraction is a clearly documented next step.

### 1:30–1:48 — Repository and onboarding

**Show:** Repository metrics, then Onboarding reading guide.

**Say:**

> CodeAtlas also surfaces measurable repository health and gives a practical reading order using READMEs, manifests, entry points, and large files—useful before asking any AI question.

### 1:48–2:22 — On-device AI proof

**Show:** Brief terminal split with `ollama list` and `/ai/status`, then AI Chat. Ask a pre-tested specific question and show the answer plus citations. Cut the waiting portion while keeping the result authentic.

**Say:**

> The core AI runs locally. BGE embeddings and FAISS retrieval stay on this machine, and the answer is generated by a quantized Phi-3 model through local Ollama—no cloud LLM API. The response is grounded in retrieved project evidence and returns file and line citations when available.

### 2:22–2:35 — Close

**Show:** Return to Galaxy or hero.

**Say:**

> CodeAtlas makes codebase onboarding visual, evidence-driven, and private. Once the models are downloaded, local-folder analysis and AI chat can run offline. The project is open source under MIT.

## Recording rules

- Do not claim Galaxy links are dependencies.
- Do not show the What-if preview unless explicitly calling it future work.
- Do not wait silently for AI generation; cut from question submission to the authentic completed response.
- Keep the final video between 2 and 3 minutes.
- Upload as unlisted YouTube or a judge-accessible Drive link, then paste the URL into the README and Unstop form.
